Namespace Phnx.Security.Algorithms

Classes

AesEncryption

An AES symmetric encryption algorithm. Suitable for password protecting documents, but not suited for storing passwords or sending data to 3rd parties. Consider using Pbkdf2Hash for storing passwords, or RsaEncryption for sending data to 3rd parties. If you're using AES with user entered passwords, you'll need to hash the passwords to a valid key length (see KeyBits). Consider using an algorithm Pbkdf2Hash for getting the password to a key of a valid size. An easy way to do this is with PasswordProtectedEncryptionService

IEncryptionExtensions

Extension for IHashWithSalt and IHashWithoutSalt

ISymmetricEncryptionExtensions

Extensions for ISymmetricEncryption

Pbkdf2Hash

A PBKDF2 hashing algorithm. Suitable for passwords, but generally too slow for checksums. Consider using Sha256Hash if you're generating checksums

RsaEncryption

An RSA asymmetric encryption algorithm

Sha256Hash

A 256 bit SHA2 hashing algorithm. Suitable for checksums, but generally not secure for passwords. Consider using Pbkdf2Hash if you're hashing passwords

SymmetricEncryption

A helper implementation of ISymmetricEncryption for using byte[] instead of for output, and other overloads

Interfaces

IAsymmetricEncryption

An asymmetric encryption algorithm where seperate keys are used for encrypting and decrypting. Suitable for sending data to 3rd parties (such as sending messages), but not suited if the same user is encrypting and decrypting the data. Consider using ISymmetricEncryption if you don't need seperate keys for encrypting and decrypting data

IEncryption

Two way encryption for encrypting and decrypting data using a key

IHashWithoutSalt

One way hashing without a salt. Useful for generating checksums for data at a fixed length

IHashWithSalt

One way hashing using a salt. Useful for storing passwords

ISymmetricEncryption

A symmetric encryption algorithm. Suitable for password protecting documents, but not suited for storing passwords or sending data to 3rd parties. Consider using IHashWithSalt for storing passwords, or IAsymmetricEncryption for sending data to 3rd parties. If you're using user created password keys instead of random keys, you'll need to hash the passwords to the algorithm's required key length first. Consider using IHashWithoutSalt for this
